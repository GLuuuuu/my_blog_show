    • keil项目结构与工程配置

    keil项目的整体结构说明：

    分层说明

    目录 作用 依赖关系
    app 用户应用逻辑 调用 driver、framework
    driver 板级硬件封装 调用 framework/driver
    framework 芯片底层支持 独立，被上层调用
    mdk 工程管理 链接所有源码
    third_lib 外部扩展功能 被 app 调用

    典型调用链（GPIO为例）

    注意事项

    • framework 下有两个 driver 文件夹，但作用不同：
      • framework/driver：STD库的芯片外设驱动（与硬件无关）
      • driver/：自己封装的板级驱动（与具体硬件相关）

    工程配置注意事项

    1. ST-Link 调试配置

    配置项 推荐设置 说明
    Port SWD 使用两线调试，节省IO口
    Clock 频率 1.8 MHz 或更低 过高会导致通信不稳定

    路径：
    Project → Options for Target → Debug → Settings（选中 ST-Link Debugger 后）
    → Port: SWD
    → Max Clock: 1.8 MHz

    Flash Download 设置

    在同一个 Settings 窗口中，切换到 Flash Download 标签页，务必勾选：✅ Reset and Run

    作用：程序下载完成后自动复位芯片并运行，不需要再手动按复位键或重新上电。

    路径：

    Debug → Settings → Flash Download → Reset and Run

    2.编译环境设置

    如果在代码中使用了 #include <stdint.h> 以及 uint32_t、int16_t 等类型，仍然报错 error: #254: type name is not allowed

    原因是 Keil 默认使用老的 C89/C90 标准。需要开启 C99 Mode

    路径：

    Project → Options for Target → C/C++ → C99 Mode

    开启后，编译器就能正确识别 stdint.h 中的类型。
