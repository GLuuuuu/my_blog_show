    • 呼吸灯

    软件模拟PWM模拟呼吸灯效果。

    一、代码实现原理

    核心思想：通过快速切换LED的亮灭，利用人眼的视觉暂留效应，调节亮和灭的时间比例（占空比）来改变亮度。

    参数理解

    一个完整周期的时间分配

    例如当 n = 300 时：

    • LED 亮：300 个空循环单位
    • LED 灭：1000 - 300 = 700 个空循环单位
    • 占空比 = 300 / 1000 = 30%

    当 n = 800 时：

    • 占空比 = 800 / 1000 = 80%，LED 看起来很亮

    二、类比PWM

    硬件PWM原理

    硬件PWM由定时器自动产生固定频率的方波，通过调节比较值（CCR）改变占空比：

    软件模拟与硬件PWM的对应关系

    硬件PWM参数 你的软件代码对应
    周期（ARR） 1000（固定的总数）
    比较值（CCR） n（亮度等级）
    占空比 n / 1000
    PWM频率 1 / (亮时间+灭时间)

    本质

    都是用“在一个固定周期内，调整高电平（亮）的时间比例”来控制亮度

    只是硬件PWM由定时器自动完成，不占用CPU；而软件方式用CPU空循环延时，执行期间CPU不能做其他事。

    三、思考总结

    代码是纯软件的PWM模拟，通过调整亮灭的时间比例来实现呼吸效果。理解了它就可以理解了PWM的核心思想：用亮/总时间的比例控制亮度。区别只是硬件PWM不用CPU去数循环，而是用定时器自动完成。
