    • VADDR.PADDR与缓存虚拟地址

    CPU 访问外设

    • 必须使用虚拟地址 (VADDR)
    • 原因：CPU 通过 MMU 发出访问，需要经过地址转换视窗

    DMA 控制器访问内存/外设

    • 必须使用物理地址 (PADDR)
    • 原因：DMA 是独立于 CPU 的总线主设备，直接操作 AXI 总线，不经过 MMU

    正确示例

    场景 使用地址类型 示例
    CPU 读写外设寄存器 非缓存虚拟地址 (VADDR) 0xbf401000
    CPU 读写内存（普通变量） 缓存虚拟地址 0x80000000
    DMA 配置源/目标地址 物理地址 (PADDR) 0x1f401000

    非缓存虚拟地址 (VADDR) —— CPU 访问外设

    • 本质：CPU 视角的“直通窗口”。地址高位的特殊标记（如 0xbf...）告诉 CPU 的 MMU（内存管理单元）：“不要缓存这次读写，直接发到总线上”。
    • 特点：绕过 Cache，每次访问都直接产生 AXI 总线事务。因此访问外设寄存器（如 FFT 控制、UART 状态）或需要实时性、无缓冲的数据时必须用它。
    • 总结：CPU 查看硬件状态的实时快照，即“专用通道”**。

    缓存虚拟地址 —— CPU 访问普通内存

    • 本质：CPU 读写内存（如 0x80000000 的 DDR）时的默认路径。数据先暂存在 CPU 内部的 Cache 中，稍后再批量写回内存，或从 Cache 中快速读取。
    • 特点：利用 Cache 加速，读写极快，但结果与总线不同步（有延迟）。这是 CPU 执行代码、操作堆栈、处理数据的标准方式。
    • 总结：CPU 日常工作的“高速缓存”，与外界（总线）有时差**。

    物理地址 (PADDR) —— DMA 访问内存/外设

    • 本质：总线上的真实地址。没有 MMU 转换，没有 Cache 干预，直接对应内存或外设的物理位置。
    • 特点：DMA 控制器（或其他总线主设备）使用它来搬运数据。因为它不经过 CPU 的 MMU 和 Cache，所以能准确找到内存中的变量或外设的数据端口。

    缓存虚拟地址是”允许使用 Cache 的普通虚拟地址”。Cache 是硬件自动管理的加速层，对软件透明，不可直接寻址。

    三种地址的关系

    地址类型 本质 比喻
    物理地址 (PADDR) 总线上的真实坐标 房屋的GPS坐标
    缓存虚拟地址 经过MMU转换 + 允许Cache 写收件地址，快递员可暂存快递柜
    非缓存虚拟地址 经过MMU转换 + 禁止Cache 写收件地址，但必须当面签收
