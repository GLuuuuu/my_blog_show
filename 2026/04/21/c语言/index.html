    一、volatile 的底层原理

    1. 使用原因：编译器优化

    现代编译器会做激进优化来提高程序运行速度。例如：

    编译器理解：”p 指向的内存没有被修改过，为什么要读两次？”，结果第二次使用的仍为第一次结果，实际想要的并不是这样。

    问题：如果 0x12345678 是硬件 FIFO，第一次读取后 FIFO （队列的感觉）会自动弹出下一个数据，第二次读取应该是不同的值（即新弹出的数据）。编译器的优化破坏了程序逻辑。

    2. 加上volatile 后的汇编层面

    3. 真实场景举例：DMA 写入缓冲区

    没有 volatile 的后果：

    CPU 可能从 Cache 读数据（认为数据未被修改），而 DMA 直接写 DRAM（CPU不知道），造成 Cache 不一致

    有 volatile 的效果：

    CPU 会绕过 Cache 或者强制刷新 Cache，从 DRAM 读取最新数据

    二、__attribute__((aligned( ))) 的底层原理

    1. 内存对齐的硬件原因

    DRAM 和 CPU 之间的数据交换不是按字节进行的，而是按总线宽度或 Cache Line。按照总线宽度交换时效率最高。如果数据跨越了两次总线传输的边界，就需要两次内存访问。
